The CSS Flexbox

With the default CSS Block model, you assign dimensions (like width and height) whether it is fixed (using px, rem, etc.) or percentage, and then use float to arrange items on the page. And this was not easy to do. You have to factor in the margins and the paddings and how much width you will have. But then, Flexbox was created.

So What Is Flexbox?

The Flexbox (also called the Flexible Box Model) is a CSS model that provides an easy and clean way to align items within a container. Flexbox is a one-dimensional layout method for arranging items in rows or columns. Flex items (or children) flex (or expand) to fill additional space or shrink to fit into smaller spaces. Flexbox is responsive and mobile friendly. Positioning Child elements is easy. The order of elements can be changed without editing the HTML. In order to implement Flexbox, if we have an element (like a div) with children, all we need to do is to set the display property to flex. 


<div class="container">
  <div class="item-1">1</div>
  <div class="item-2">2</div>
  <div class="item-3">3</div>
</div>



.container {
  display: flex;
}

In the above code and illustration, the boxes were also given a height and a width of 100 pixels. We can see that by setting the display as flex on an element, we are setting that element (the div item with dashed border) as a flex container. All the direct children inside that div element then become flex items. By default, if you specify no other properties, the flex items will line up in a row from left to right and will take up only as much space as they need, or in this case based on the given width. Items will also automatically shrink down to fit within the flex container if they would normally overflow.

Also, take note that the numbers 1, 2, and, 3 are centered inside the blue boxes because the the children boxes were also given a diplay property of flex.

Flexbox Layout

Flexbox was the first CSS layout method that worked in a completely different way than normal CSS. Instead of worrying about block/inline elements, the flexbox model worries about a main axis (the dark blue horizontal line below) and a cross axis (orange vertical line).

Main Axis
Cross Axis

By default the main axis (shown in dark blue) goes horizontally across the container and the cross axis goes vertically. This means that any layout method that deals with the main axis will layout elements horizontally while the cross axis will layout elements vertically. Let's look at how we can layout elements along the main axis first.

justify-content

For all these examples we will assume that the flex-items have a width of 20%.


.flex-container {
display: flex;
}
flex-start (default)

If you don't set the justify-content, the default value will be flex-start. It will place all the items at the "start" of the main axis. The start is at the left side of the main axis by default. As you can see below:

.flex-container {
  display: flex;
  justify-content: flex-start;
}
flex-end

Flex-end places all items at the end of the main axis which is the right of the axis by default.


.flex-container {
  display: flex;
  justify-content: flex-end;
}

center

Center places all items at the center of the main axis.


.flex-container {
  display: flex;
  justify-content: center;

space-between

Space-between takes all the extra space of the flex container and evenly distributes them to space the items as far as possible from each other while filling the full width of the container (or height if the main axis is vertical).


.flex-container {
  display: flex;
  justify-content: space-between;
}

space-around

This is very similar to space-between space-between, but it also adds space on both the left and right edges of the flex container. This means that all the children or flex items will have spaces on both sides.


.flex-container {
  display: flex;
  justify-content: space-around;
}
