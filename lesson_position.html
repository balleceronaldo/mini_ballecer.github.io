Static Positioning

If you inspect from the developer tools, static is the default position for all the elements. Static essentially means that the elements will follow according to how they were structured in the html file. For example, child one is under the parent and child two follows child one.

Relative Positioning

The Relative position works almost exactly the same as the static position. If you change the position of the first child to relative, nothing will happen but this will allow you to change the top, left, right and bottom of the first child element.

Now, let's see what happens if we changed the second child to relative position and give it a left value of 10px.

We can see that the first child overflows the second child. What is happening here is that the first child was removed from the static flow (although since some parts are still occupied by the first child, the second child cannot occupy it yet) where the elements would push the other elements if the positions where changed. In general, relative position is not used for top, left, right, and bottom positioning.

The Absolute Position

If we changed the first child position to absolute without any other changes from our first diagram:


.parent {
  background-color: lightblue;
  padding: 10px;
}
.child-1 {
  background-color: lightgreen;
  position: absolute;
}

.child-2 {
  background-color: orange;
}

.child-3 {
  background-color: lightpink;
}
Parent
One
Two
Three

We can see that the second child is behind the first child and taken its place but also the first child occupies only the width of the text (the inner text inside the div). The first child is completely out of the static flow.

Position absolute allows you to position the element relative to the body of the document which in this case is the top-left corner of the screen. If we give the first child the value of position absolute and top value of 10 pixels:


.parent {
  background-color: lightblue;
  padding: 10px;
}
.child-1 {
  background-color: lightgreen;
  position: absolute;
  top: 3600px;
}

.child-2 {
  background-color: orange;
}

.child-3 {
  background-color: lightpink;
}

You can see that the first child is just a few pixels above the diagram but actually is 3600 pixels from the top of the screen.

If we use position relative to the parent div:


.parent {
  background-color: lightblue;
  position: relative;
  padding: 10px;
}
.child-1 {
  background-color: lightgreen;
  position: absolute;
  top: 30px;
}

.child-2 {
  background-color: orange;
}

.child-3 {
  background-color: lightpink;
}

We can see that the absolutely positioned element is looking for a relatively positioned element to align with and if it doesn't find one, it will position itself to the very top of the screen. We can also observe that the absolute positioning will follow the padding of the relatively positioned parent if followed.
